# HLS Web 青春活力配色改版设计

## 背景

`hls-web/` 当前前端界面采用深色科技风，视觉上更接近运维或数据后台。当前系统面向初中生使用，并交由校方管理，界面需要在保持清晰、可信、易读的前提下，显著提升亲和力、青春感和校园氛围。

本次改动仅针对视觉主题与色彩系统，不调整业务逻辑、页面结构和交互流程。

## 目标

1. 将整站主题从深色科技风调整为浅色、明亮、友好的校园管理风格。
2. 同时兼顾两类使用者的感受：
   - 对学生端观感更轻松、有活力、不压抑。
   - 对校方使用场景仍保持专业、稳定、清晰。
3. 在不改动页面结构的前提下，通过颜色、边框、阴影和背景层次重建视觉语言。

## 非目标

1. 不修改 Vue 业务逻辑。
2. 不新增页面或组件。
3. 不调整信息架构、字段布局或交互流程。
4. 不引入新的设计系统依赖或主题框架。

## 当前实现约束

1. 样式主要集中在 `hls-web/src/style.css`。
2. 页面入口简单，`src/main.js` 仅加载全局样式和 `App.vue`。
3. 当前样式已经使用了一组全局 CSS 变量，适合直接替换语义色板并调整主要组件样式。

## 设计方向

### 总体风格

- 风格定位：活力专业型
- 页面基调：浅色主界面
- 关键词：校园感、清爽、明亮、亲和、可信、现代

### 视觉原则

1. 保留管理系统应有的清晰层级和状态识别。
2. 避免过度卡通化或儿童化，保证校方接受度。
3. 通过浅背景、柔和卡片、亮色主按钮来体现青春感。
4. 点缀色只用于强化重点，不大面积使用，防止界面杂乱。

## 配色系统

### 主色

- 主色：`#3B82F6`
- 主色高亮：`#60A5FA`

用途：
- 主按钮
- 焦点态
- 重要链接
- 核心高亮标签

### 辅助色

- 薄荷绿：`#34D399`

用途：
- 成功状态
- 推荐、完成、健康状态
- 局部正向反馈

### 点缀色

- 珊瑚橙：`#FF8A65`

用途：
- 强提醒
- 轻量高亮
- 局部视觉活跃点

### 警示色

- 危险色：保留偏红语义，但改为更适合浅色界面的清晰危险色
- 警告色：使用更柔和的暖黄色，避免刺眼

### 中性色

- 页面底色：`#F6F9FF`
- 次级背景：浅蓝白渐变
- 卡片底色：`#FFFFFF` 或接近 `rgba(255, 255, 255, 0.88)`
- 主文字：`#1F2A44`
- 次文字：`#60708F`
- 边框：低饱和浅蓝灰

## 组件级改动范围

### 1. 页面背景

将当前深蓝渐变背景替换为浅米白到浅蓝的轻渐变背景，保留少量柔和光斑，以维持页面的活力和层次，但不再表现深色科技感。

### 2. Hero 区域

保留头部大卡片结构，但从深色玻璃面板切换为浅色卡片。

预期变化：
- 背景改为浅色渐变
- 边框更轻
- 阴影更柔和
- 装饰气泡改为蓝、绿、珊瑚橙的低透明度版本

### 3. Panel / Card

所有主要面板由深色半透明背景切换为浅色卡片，保留圆角与阴影层级，降低压迫感，提高长时间阅读舒适度。

### 4. 输入框

输入框改为浅底深字，焦点态使用亮蓝描边和柔和 focus ring，保证表单清晰可读。

### 5. 按钮

- 主按钮：亮蓝渐变或亮蓝实色，文字使用深色或高对比白色，以实际对比效果为准
- 次按钮：浅底描边
- 危险按钮：浅红底加红色边框
- 微按钮与标签按钮：统一到浅色体系

### 6. 状态标签与提示条

- 成功：薄荷绿体系
- 错误：浅红体系
- 警告：暖黄体系
- 普通信息：浅蓝体系

要求不同状态在浅背景下保持足够区分度，且不只靠颜色传达信息。

### 7. 进度与图表线条

进度条、sparkline、局部高亮线条统一到新的亮蓝/薄荷绿体系，避免保留旧版深色科技蓝紫组合。

## 可访问性与可用性要求

1. 正文和主要标签在浅背景上的对比度满足可读性要求。
2. 输入框、按钮和可点击元素保留清晰焦点态。
3. 状态颜色不能过浅，避免在投影设备或普通办公显示器上失真。
4. 不能只用颜色表达状态，现有图标、文案和状态文字符号需要继续保留。

## 实施方案

### 文件范围

- 主要修改文件：`hls-web/src/style.css`
- 预期无需修改：`hls-web/src/App.vue`、`hls-web/src/main.js`

### 实施方式

1. 先重写 `:root` 中的全局变量。
2. 再批量调整背景、面板、输入框、按钮、状态条、进度条等关键样式。
3. 仅在存在明显硬编码颜色时做定向替换，尽量收敛到语义变量。

## 验收标准

1. 首屏观感明显从“深色科技后台”转为“浅色校园管理系统”。
2. 界面整体更年轻、更友好，但不失正式感。
3. 按钮、输入框、卡片、状态条视觉统一。
4. 页面可读性优于当前版本，特别是长时间浏览和录入场景。
5. `vite build` 可以通过。

## 风险与注意事项

1. 当前样式文件存在较多硬编码颜色，改版时需要注意遗漏点，避免新旧配色混用。
2. 主按钮文字颜色需要在最终实现时实际验证对比度，避免渐变背景下可读性不足。
3. 若局部模块仍保留深色透明背景，会破坏整体统一性，需要一并调整。
